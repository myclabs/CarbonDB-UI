Contribute

Your contributions are welcomed, this projects depends on them. Since it is altogether open data and open source, you are welcomed to contribute either to the data or to the code.

Contribute to the data

Source data are maintained by My C-Sense as an open source project on Github. Here are the basic ways to contribute:

  1. comment existing issues,
  2. submit an issue,
  3. submit an improved version of the ontology.

The first two ways are rather simple, you will just need an account on Github (leading platform of code hosting).

In order to contribute the third way you will have to directly edit the CarbonDB source data. Those take the form of a domain ontology expressed in the universal Ressource Description Framework (RDF) language (a refinement of XML specified by World Wide Web Consortium).

These source data can be viewed online and downloaded from the project repository on Github. They are contained in a single file: "carbondb.rdf".

Once you have downloaded the source data file, you will be able to edit them with any ontology editor, for instance Protégé (this is the one we used from the very beginning), and finally submit the improved version as a pull request.

Contribute to the code

Underlying code is made of two layers: an inference engine ("reasoner") and a front-end web application. Only the inference engine is open-source at this stage. Its main role is to compute secondary data (in particular elementary flows of almost all processes) from source data (coefficients).

Please join us on the repository of inference engine on Github.
